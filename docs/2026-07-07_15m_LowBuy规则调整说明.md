# 2026-07-07 15m LowBuy 规则调整说明

更新时间: 2026-07-07 09:30:51 CST

## 背景

15m 主系统当前主要运行 LowBuy 策略，反转策略已关闭，Fair Value 仅保留监控/记录用途，不参与当前生产开仓决策。

本次调整来自对当前 48 笔模拟流水的复盘。最新亏损单显示，盘口从接近 50/50 快速变化为 `Up 71c / Down 30c` 时，旧规则把 `Down 30c` 当作低买机会买入，随后价格继续下跌并触发 5 分钟止损。这个场景更像强趋势破位中的弱势腿，而不是正常震荡回归。

核心问题: 旧规则只看价格是否落在区间内，缺少“趋势破位保护”和“瞬时跳水过滤”。

## 本次改动

1. 收紧 LowBuy 入场价格带

旧规则:

```text
30c - 38c
```

新规则:

```text
32c - 34c
```

目的: 避开 `30c - 31c` 这类刚破位的弱势腿，同时不再继续买 `35c - 38c` 这段历史表现较差的区间。

2. 增加对侧过强过滤

新增规则:

```text
如果对侧 ask >= 68c, 当前候选侧不买
```

例子:

```text
Up ask = 70c
Down ask = 33c
```

旧规则可能买 Down，新规则会跳过。原因是对侧已经明显占优，候选侧低价更可能是强趋势压制，不是便宜错价。

3. 增加瞬时跳水过滤

新增规则:

```text
如果候选侧 ask 在最近 20 秒内下跌 >= 8c, 不买
```

例子:

```text
Down ask: 50c -> 33c
```

这种情况会被视为破位下跌，不再接飞刀。

4. 增加提前硬止损

旧规则主要等到剩余 5 分钟时才做时间止损。

新增规则:

```text
如果 best_bid <= entry_price * 55%, 立即硬止损
```

例子:

```text
entry = 33c
hard stop threshold = 18.15c
bid <= 18.15c 时直接离场
```

目的: 减少 `30c -> 3c` 这类大亏，不再机械等到 5 分钟止损。

5. 增加开仓复盘字段

后续新开仓会在交易流水中记录更多入场上下文:

```text
entry_mte
entry_bid
entry_ask
opposite_ask
direction_hint
obi
recent_ask_drop
```

目的: 以后复盘时可以判断一笔单是正常震荡入场，还是接到了趋势破位。

6. 前端增加硬止损标签

新增 `hard_stop` 显示映射:

```text
硬止 / 硬止损
```

7. 修复重启脚本无法杀掉 daemon 化 bot 的隐患

问题:

```text
start_paper_sim.sh / stop_paper_sim.sh 过去只发一次 TERM 并等待 1 秒
daemonize.py double-fork 后 bot 的 PPID=1
bot 没有 graceful shutdown 逻辑时, 旧进程可能继续存活
```

风险:

```text
重启后老 bot + 新 bot 可能同时跑一段时间
会双写 paper_trade_state.json 和 position_audit.jsonl
```

修复:

```text
stop_pid_file 现在统一走 TERM -> 最多等 5 秒 -> 仍存活则 KILL
stop_port_listener 也复用同样的强制回收逻辑
```

## 后续观察重点

1. 新规则下开仓频率是否明显下降。
2. `32c - 34c` 区间是否比旧规则质量更高。
3. 是否还会出现对侧已经大于 `68c` 时买入弱势腿。
4. Fair Value 面板是否仍然只作为观察信息，而没有被误接回 LowBuy 主链路。
4. `hard_stop` 是否有效减少大亏单。
5. 新增复盘字段是否足够判断每笔单的入场质量。
